## Changes Since Last Benchmark

- Lazy per-block decode: replaced eager full-column decode (96MB flat long[]) with on-demand per-block (128KB) decode using AtomicReferenceArray
- FilterRewrite DV fallback for aggregation acceleration

## Key Findings

- Lazy decode improved force-merge sort significantly (desc-sort-ts-after-FM: +459% → +134%) but sort operations remain the primary bottleneck
- DocValuesSkipper is never called — FieldInfo.docValuesSkipIndexType=NONE for non-skiplist fields. Forcing RANGE was tried but hurt aggregation performance and was reverted.
- Sort with `track_total_hits=true` (default) visits all docs regardless of skipper — the fundamental bottleneck
- Parquet block decode (Zstd decompression + bit unpacking per 16K-doc block) is inherently slower than Lucene90's zero-copy mmap DirectReader for random access
