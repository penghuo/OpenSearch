### Architectural Blockers

1. **Force-merge DESC sort** — Single-segment reverse iteration on Parquet-backed DocValues triggers extreme latency (LUCENE-7482). The +49789% p90 on desc-sort-timestamp-after-force-merge-1-seg (3621ms vs 7ms) indicates a degenerate code path.
2. **Scroll** — DerivedSource reconstruction adds ~2x overhead per scroll page. Fundamental to the _source-less design.
3. **p90 tail latency** — match-all (+392% p90) and status-400s-in-range (+6446% p90) show extreme variance despite moderate svc50 regressions, suggesting cold-path or GC-induced spikes.
4. **GC pressure** — 3x more young gen collections with Parquet, likely from block decompression allocations.

### Optimizations Applied

- FilterRewrite DocValues fallback for aggregation acceleration
- Hybrid codec: Lucene90 for numeric/date sort fields, Parquet for others
- ApproximateMatchAllQuery for match-all optimization
- DerivedSource batch pre-fetch for scroll
- Zstd compression level 9
- Eager segment warmUp (lazy causes p90 spikes)
