## Notes

- Previous parquet run with `materialize()` OOM'd at 30.6GB heap during first query. Removed `materialize()` to fix.
- Storage reduction 41% comes from disabling `_source` (parquet uses DerivedSource).
- Aggregations pass or improve (hourly_agg_with_filter -13%, multi_term_agg -25%).
- Sort + _source fetch operations regress due to DerivedSource overhead scaling with doc count.
- `desc/asc_sort_with_after_timestamp` regressed 330-582% — DerivedSource is the dominant bottleneck.
- `scroll` +381% — each scroll page reconstructs _source for 10 docs via DerivedSource.
